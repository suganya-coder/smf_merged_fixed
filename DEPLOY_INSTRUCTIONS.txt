NEW API ENDPOINTS (all require JWT token)
──────────────────────────────────────────────────────────────
  GET /api/edudrill/overview
      → Institution-wide KPIs (students, faculty, attendance %)

  GET /api/edudrill/depts
      → All department cards with live stats

  GET /api/edudrill/dept/{dept_key}
      → Single dept drill: courses, sections, faculty list
      Example: /api/edudrill/dept/CS

  GET /api/edudrill/dept/{dept_key}/course/{course_key}/section/{section}/students
      → Full student list for a section with attendance %
      Example: /api/edudrill/dept/CS/course/DS/section/A/students

  GET /api/edudrill/dept/{dept_key}/faculty
      → Faculty list for a department with attendance stats

HOW THE DRILL-DOWN WORKS
──────────────────────────────────────────────────────────────
  Home  →  All departments (cards with KPIs + 4 charts)
    ↓
  Dept  →  All courses in dept + section table + faculty + 3 charts
    ↓
  Section  →  Full student list + attendance %, sortable/filterable
              3 charts: status split, distribution, top vs bottom 5

DATA SOURCES (all real — no mock data)
──────────────────────────────────────────────────────────────
  • students table      → enrollment counts per section
  • attendance table    → live present/absent, 30-day avg %
  • faculty table       → faculty list per dept
  • faculty_attendance  → faculty attendance % (30-day)

Dept taxonomy (which section = which dept) uses the same
DEPT_META structure already in api_features.py, so it stays
in sync with your existing Departments drill-down page.
